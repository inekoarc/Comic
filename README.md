# 漫画管理器

一个本地漫画库管理和阅读工具。服务端扫描本机漫画文件夹，前端提供漫画库、排行榜、分类、详情页和阅读器。评分、标签、观看次数、简介、分类封面等用户数据会保存到项目内的 `data/metadata.json` 和 `data/config.json`。

## 启动

```bash
npm start
```

然后打开：

```text
http://127.0.0.1:9000
```

Windows 下也可以双击 `ComicLauncher.exe` 启动，详见 [LAUNCHER.md](LAUNCHER.md)。

如果部署到服务器并希望从局域网/公网访问，可设置监听地址：

```powershell
$env:HOST="0.0.0.0"
npm start
```

## 漫画目录结构

推荐按一级分类、二级漫画文件夹组织：

```text
漫画根目录/
  分类A/
    漫画1/
      001.jpg
      002.jpg
    漫画2/
      001.png
      002.png
  分类B/
    漫画3/
      001.webp
```

系统会把一级目录识别为分类，把包含图片的文件夹识别为漫画。若漫画目录中存在 `元数据.json`，会尝试初始化标签和简介。

## 主要功能

- 漫画库按更新时间从新到旧展示，支持分类筛选和搜索。
- 自动识别同一作品合集，详情页支持按名称/更新时间、正序/倒序排列章节。
- 分类页只展示二级目录，可自定义分类封面，并提供“随机”入口。
- 阅读器支持下拉和单页模式、自动滚动、自适应宽度、阅读进度提示。
- 支持删除整部漫画、删除当前图片页、移动漫画到其他分类，并同步移动本地文件夹。
- 同步扫描时显示全局进度条，移动漫画时会避开正在扫描的状态。
- 排行榜支持按评分和观看次数排序，每页渲染 50 条。
- 支持从标题中初始化作者标签，从 `元数据.json` 初始化标签和简介。

## 本地数据

- `data/config.json`：漫画根目录和分类封面配置。
- `data/metadata.json`：评分、标签、观看次数、简介等用户数据。
- `data/library-index.json`：扫描生成的漫画索引。
- `data/thumbs/`：缩略图缓存。

以上文件会按本机环境生成，默认不会提交到 Git 仓库。

## 项目文件

- `index.html`：页面结构。
- `styles.css`：界面样式和响应式布局。
- `app.js`：前端状态、交互和渲染逻辑。
- `server.js`：本地 API、静态服务、目录扫描和文件操作。
- `tools/ComicLauncher.cs`：Windows 启动器源码。
- `build-launcher.ps1`：重新构建启动器的脚本。
